## Jenkins란?
Jenkins는 오픈 소스 기반의 자동화 서버로, 소프트웨어 개발 과정에서 빌드·테스트·배포를 자동화하여 코드 변경 사항을 지속적으로 통합하고 배포할 수 있도록 지원하는 도구  
이를 통해 개발자들은 코드 변경 사항의 품질을 신속히 확인하고 문제를 조기에 발견 가능

## CI/CD
지속적인 통합과 배포를 통해 개발 프로세스의 효율성과 안정성을 높이는 DevOps의 핵심 요소
- **지속적인 통합(CI)**: 개발자들이 자주 코드를 병합하고 자동화된 테스트를 통해 통합 문제를 조기에 발견
- **지속적인 배포(CD)**: 코드 변경 사항을 빠르게 배포하여 사용자에게 전달하는 것을 의미

Jenkins에서는 Pipeline as Code(Jenkinsfile)를 통해 CI/CD 과정을 정의하며, 빌드부터 테스트, 배포까지의 전체 흐름을 코드로 관리할 수 있음

## Jenkins의 장점
 - 다양한 플러그인 지원
- 사용자 친화적인 인터페이스
- 빌드, 테스트, 배포 작업이 자동화되어 개발 효율성이 크게 향상
- Jenkins는 SonarQube, OWASP Dependency-Check 등 보안 도구를 파이프라인에 쉽게 통합할 수 있어 개발부터 배포까지 전 과정에서 자동화된 보안 검사를 수행

## 컨테이너화 및 쿠버네티스 통합
컨테이너화 및 쿠버네티스 통합이란 애플리케이션을 독립적인 컨테이너로 패키징하여 쿠버네티스 플랫폼에서 오케스트레이션하는 것을 의미  
Jenkins는 CI/CD 파이프라인을 담당하고, 쿠버네티스는 컨테이너화된 애플리케이션의 배포·확장·복구를 담당함으로써 역할을 분리하여 운영 가능  
이를 통해 환경 간 일관성, 신속한 배포 및 롤백, 자동 확장 등 다양한 이점을 누릴 수 있음  

## Jenkins 사용법
1. Jenkins 설치 및 초기 설정
2. CI/CD를 위한 프로젝트 생성 및 구성
    - 효율적인 코드 관리를 위해 GitHub 등 사용중인 소스 코드 관리 도구와 연동
    - 소스 코드 변경 시 자동으로 빌드가 실행되도록 빌드 트리거 조건을 설정
    - 신속한 문제 해결을 위해 빌드 후 작업으로 빌드 실패 시 이메일이나 Slack 등을 통해 개발자에게 즉시 알림이 가도록 설정
3. 빌드 확인

## Jenkins 사용 시 주의 사항
1. 보안 설정: 관리자 계정 생성, 강력한 비밀번호 정책 적용, 역할 기반 접근 제어 설정이 필요
2. 필요한 플러그인만 설치하고 정기적으로 업데이트
3. 적절한 리소스 할당과 병렬 실행 설정, 불필요한 빌드 최소화 등을 통해 빌드 큐를 효율적으로 관리해야 함
