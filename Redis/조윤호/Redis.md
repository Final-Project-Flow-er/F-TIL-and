# 1. Redis란?
>오픈 소스 기반의 인 메모리(In-Memory) 기반의 Key-Value 저장소이다.

---

# 2. In-Memory
* 디스크가 아닌 RAM에서 데이터를 처리하기 때문에 속도가 매우 빠름

---

# 3. 영속성 옵션
## 3.1 AOF(Append Only File)
>데이터가 바뀌는 모든 명령을 순서대로 기록해두는 로그 파일

* Redis는 기본적으로 메모리에 존재하므로 서버 꺼지면 데이터가 사라지므로 복구하기 위해 사용하고 복구 정확도가 높음

---

## 3.2 Snapshot

>일정 주기마다 메모리 내 데이터를 그대로 파일로 저장하는 방식

* 위에서 말한 것 처럼 메모리에 존재하므로 서버 꺼지면 데이터가 사라지므로 복구하기위해 사용하고 빠르게 저장하고 빠르게 복구가 가능

# 4. Pub/Sub
>메시지를 발행(Publish) 하는 주체와 메시지를 구독(Subscribe) 하는 주체를 분리하여 실시간 메시지를 전달하는 시스템

## 4.1 동작과정
```
Publisher(발행자) --PUBLISH 메세지--> Redis Server --> Subscriber1(구독자1),Subsccriber2(구독자2),,, 
```

* 메세지는 서버에 저장되지 않으며, 구독 중인 클라이언트에게 즉시 전달된다.
