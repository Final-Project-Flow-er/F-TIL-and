## DNS
DNS(Domain Name System)란 사람이 이해하기 쉬운 도메인 이름을 IP 주소로 변환해주는 시스템
### 특징
1. 도메인 네임의 계층적 구조
2. 도메인 네임을 관리하는 네임 서버의 계층적 구조
3. 분산 구조로 설계되어 장애에 강함

## Gateway
게이트웨이는 클라이언트와 오리진 서버 사이에서 요청을 받아 오리진 서버로 전달하는 중간 서버

- 프로토콜 변환, 라우팅, 인증 등의 역할 수행
- API Gateway 형태로 많이 사용됨

## Proxy
프록시란 클라이언트가 선택한 메시지 전달의 대리자로, 주로 캐시 저장, 클라이언트 암호화 및 접근 제한 등의 기능을 제공
### 주요 기능
- 캐싱
- 보안(암호화, 접근 제한)
- 로드 밸런싱
- 익명성 보장
### 종류
- 포워드 프록시: 클라이언트와 가까운 위치에 있으며, 클라이언트의 요청을 받아 서버로 전달하는 역할
- 리버스 프록시: 오리진 서버와 가까운 위치에 있으며, 클라이언트의 요청을 받아 오리진 서버의 응답을 클라이언트에 전달하는 역할

## CORS
CORS는 브라우저의 보안 정책으로, 서로 다른 출처간의 리소스 요청을 제한하는 메커니즘
- Origin = Protocol + Host + Port
- 서버가 응답 헤더로 허용 여부를 명시함

## 세션과 쿠키
### 세션
- 서버 측에 저장되는 사용자 상태 정보
- 클라이언트는 세션 ID만 쿠키로 보관
- 서버 메모리 또는 Redis 등에 저장됨
### 쿠키
- HTTP의 stateless 특성을 보완하기 위한 수단
- 서버에서 생성되어 클라이언트에 저장
- <이름, 값> 형태의 데이터

## JWT(Json Web Token)
JWT는 사용자 인증 정보를 JSON 형태로 안전하게 전달하기 위한 토큰 기반 인증 방식
### 특징
- Stateless 인증
- 서버에 세션 저장 불필요
- 토큰 탈취 시 보안 위험 → 만료 시간, HTTPS 필수

## RESTful API
REST 원칙을 따르는 API 설계 방식
### 주요 원칙
- 자원 중심 설계
- HTTP Method 사용 (GET, POST, PUT, DELETE)
- Stateless
- URI는 명사, 동사는 Method로 표현
