## OSI 7계층
네트워크 통신 과정을 7단계로 나눈 표준 모델
- **물리 계층**: 비트 신호
- **데이터 링크 계층**: 같은 LAN에 속한 호스트끼리 통신 (MAC주소 사용)
- **네트워크 계층**: 네트워크 간 통신 (IP주소 사용)
- **전송 계층**: 신뢰성 있는 전송을 가능하게 함 (포트 번호 사용, TCP, UDP등이 있음)
- **세션 계층**: 응용 프로그램 간의 연결 상태를 의미하는 세션을 관리
- **표현 게층**: 인코딩과 압축, 암호화 등을 수행
- **응용 게층**: 여러 네트워크 서비스 제공

## TCP와 UDP
- **TCP**: 신뢰할 수 있는 통신, 연결형 통신 -> 상태관리, 흐름 제어, 오류 제어, 혼잡 제어 제공 O / 연결 수립, 종료 과정 O
- **UDP**: 신뢰할 수 없는 통신, 비연결형 통신 -> 상태관리, 흐름 제어, 오류 제어, 혼잡 제어 제공 X / 연결 수립, 종료 과정 X
패킷의 유실 없는 송수신을 원한다면 TCP 선택, 빠른 송수신을 원한다면 UDP 선택

## TCP의 연결 수립: three-way handshake
1. **A -> B**: SYN 세그먼트 전송 (A 순서 번호 포함)
2. **B -> A**: SYN + ACK 세그먼트 전송 (B 순서 번호 + 1에서 보낸 세그먼트에 대한 확인 응답 번호 포함)
3. **A -> B**: ACK 세그먼트 전송 (A 순서 번호 + 2에서 보낸 세그먼트에 대한 확인 응답 번호 포함)

## TCP의 오류, 흐름, 혼잡 제어
- **오류 제어**: 송수신 과정에서 잘못 전송된 세그먼트가 있을 경우, 이를 재전송하여 오류를 제어 (중복된 ACK 세그먼트가 도착했을 때, 타임아웃이 발생했을 때)
- **흐름 제어**: 수신 호스트가 한 번에 받아 처리할 수 있을 만큼(수신 윈도우 크기)만 전송
- **혼잡 제어**: 혼잡 없이 전송할 수 있을 정도의 양(혼잡 윈도우)만큼만 송신

## HTTP
### HTTP의 특징
1. 요청 응답 기반 프로토콜
2. 미디어 독립적 프로토콜
3. 스테이트리스 프로토콜 (서버는 HTTP 요청을 보낸 클라이언트 관련 상태를 기억하지 않음)
4. 지속 연결 프로토콜

### HTTP 메서드
`GET`(조회), `POST`(생성), `PUT`(수정), `DELETE`(삭제), `PATCH`(부분 수정) 등

### HTTP 상태 코드
- 100번대: 정보
- 200번대: 성공 (200 OK, 201 Created, 202 Accepted, 204 No Content)
- 300번대: 리다이렉션
- 400번대: 클라이언트 요청 오류 (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 405 Method Not Allowed)
- 500번대: 서버 내부 오류 (500 Internal Server Error)
