# 스프링 배치란?
대용랑 데이터를 처리하기 위한 프레임 워크로, 스프링 프레임 워크 기반에서 작동한다.
일반적으로 배치 작업은 대량의 데이터를 처리하거나, 주기적이고 반복적인 작업을 실행하는 데 사용되며, 스프링 배치는 이러한 작업을 효율적이고 안정적으로 처리할 수 있는 기능들을 제공한다.
- 로깅 및 추적
- 트랜잭션 관리
- 작업 처리 통계
- 작업 재시작
- 건너뛰기
- 리소스 관리

# Batch와 Scheduler의 차이
배치는 논리적 또는 물리적으로 관련된 일련의 데이터를 그룹화하여 일괄 처리하는 방법을 의미한다.
반면 스케쥴러는 주어진 작업을 미리 정의된 시간에 실행할 수 있게 해주는 도구나 소프트웨어를 의미한다.

여기서 주의할 점은 배치는 대량의 데이터를 일괄적으로 처리할 뿐, 특정 주기마다 자동으로 돌아가는 스케쥴링과는 관련이 없다는 것이다.
Spring Batch는 스케쥴러와 함께 사용할 수 있도록 설계되어 있을 뿐이지 스케쥴러 자체를 대체하지는 않는다.
따라서 작업 스케쥴링 라이브러리인 Quartz 등과 Spring Batch를 비교하는 것은 적절치 못하다.

# Batch 사용 사례
## 일매출 집계
커머스 사이트의 경우 하루에 거래건이 50만에서 100만 건까지 이루어지는데 관련 데이터는 최소 100만에서 200만 행 이상이 될 수 있다.
한 달 동안은 이런 데이터가 5000만에서 1억까지 쌓일 수 있다.
이런 데이터를 실시간으로 집계하는 쿼리를 실행하면 조회 시간이 길어지고 서버에 많은 부담이 가게 된다.
그래서 매일 새벽에 전날의 매출 집계 데이터를 미리 생성해두어, 외부에서 요청이 오면 전날에 집계한 데이터를 제공하는 방법으로 해결한다.

## 데이터 백업
대규모 데이터 베이스를 운영하게 되면, 데이터의 일관성을 보장하기 위해 주기적인 데이터 백업이 필수적이다.
그러나 이런 배겁 작업은 시스템에 상당한 부담을 줄 수 있기에 일반적으로 사용자 트래픽이 상대적으로 적은 시간대에 배치 처리 방식을 통해 백업 작업을 실행한다.
