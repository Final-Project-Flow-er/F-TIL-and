# Virtual Memory

---

> 운영체제가 물리 메모리의 한계를 극복하기 위해 사용하는 메모리 관리 기법

---
# 1. 핵심 아이디어
## 1.1. 보조기억장치 활용
RAM이 꽉 차면 당장 쓰지 않는 데이터는 하드디스크나 SSD의 일부 공간(스왑 영역)으로 잠시 옮겨둠
빈 RAM 공간에 지금 당장 필요한 데이터를 채워넣음
## 1.2. 부분 적재
프로그램을 실행할 때 전체를 한 번에 메모리에 올리지 않고
지금 당장 필요한 부분(Page)만 잘라서 올림
나머지는 보조기억장치에 둠

---
# 2. 가상 메모리의 주요 기능
가상 메모리는 단순히 용량만 늘려주는 것이 아니라 시스템 안정성과 보안에도 중요
## 2.1. 메모리 확장
물리 메모리보다 큰 프로그램을 실행할 수 있게 해줌
## 2.2. 프로세스 격리(보호)
각 프로세스에게 독립적인 가상 주소 공간을 부여
프로세스 A가 프로세스 B의 메모리를 침범하거나 훔쳐보는 것을 원천적으로 차단하여 시스템 안정성을 높임
개발자는 물리 메모리의 실제 위치를 신경 쓸 필요 없이 자신만의 0번지부터 시작하는 연속된 메모리 공간을 쓴다고 착각하게 만듬

---
# 3. 동작 원리(페이징 & 페이지 폴트)
가상 메모리는 MMU(Memory Management Unit)라는 하드웨어의 도움을 받아 작동
## 3.1. 가상 주소 사용
CPU는 가상 주소를 사용해 데이터 요청
## 3.2. 주소 변환
MMU가 페이지 테이블을 보고 가상 주소를 실제 물리 주소로 변환
## 3.3. 페이지 폴트
만약 찾는 데이터가 현재 RAM에 없다면 페이지 폴트라는 인터럽트 발생
## 3.4. 스왑
운영체제는 하던 일을 멈춤
디스크에서 해당 데이터를 찾아 RAM으로 가져옴
이 때 RAM이 꽉 차 있다면 안 쓰는 데이터를 디스크로 내보냄
