# Demand Paging

---

> [[Virtual Memory]] 관리의 핵심 전략으로 프로그램 실행에 필요한 페이지만을 그때그때 메모리에 적재하는 기법

---
# 1. 핵심 원리
## 1.1. Lazy Loading(게으른 적재)
프로그램을 실행할 때 운영체제는 물리 메모리에 아무것도 올리지 않거나
아주 기본적인 페이지만 올린 채 시작
나머지 데이터는 보조기억장치에 그대로 둠
## 1.2. Page Fault(페이지 폴트) 처리
CPU가 특정 데이터에 접근하려 할 때 해당 페이지가 메모리에 없다면 페이지 폴트 인터럽트 발생
운영체제는 하던 일을 멈추고 디스크에서 해당 페이지를 찾아 메모리의 빈 공간(프레임)으로 가져옴
**순수 요구 페이징(Pure Demand paging)
- 아예 메모리에 페이지가 하나도 없는 상태에서 시작
- 첫 명령어를 실행하자마자 페이지 폴트를 일으키며 하나씩 채워가는 극단적인 방식

---
# 2. 장단점
## 2.1. 장점
### 2.1.1. 메모리 절약
당장 안 쓰는 데이터는 메모리를 차지하지 않음
더 많은 프로그램을 동시에 실행 가능
### 2.1.2. 빠른 시작
프로그램 전체를 로딩할 필요가 없으므로 실행 반응 속도가 빠름
## 2.2. 단점
### 2.2.1. 초기 오버헤드
프로그램 실행 초반에는 필요한 페이지를 계속 가져와야 하므로 페이지 폴트가 자동으로 발생해 느릴 수 있음
### 2.2.2. Thrashing 위험
메모리가 너무 꽉 차서 페이지를 넣고 빼는 작업(Swapping)만 하느라 실제 일은 못 하는 상황이 발생 가능

---
# 3. 페이지 교체(Page Replacement)
요구 페이징을 하다 보면 언젠가 메모리는 가득 참
이 때 새 페이지를 가져오려면 기존 페이지 중 하나를 쫓아내야 함(Swap-out)
누구를 쫓아낼 것인지 결정하는 것이 페이지 교체 알고리즘
