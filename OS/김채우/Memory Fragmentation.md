# Memory Fragmentation 

---

> 운영체제의 메모리 할당과 해제 과정이 반복되며 잔여 메모리가 충분해도 공간이 흩어져 있어 제대로 사용하지 못 하는 현상

---
# 1. 종류
## 1.1. 내부 단편화(Internal Fragmentation)
### 1.1.1. 상황
프로세스에게 필요한 양보다 더 큰 메모리 공간을 할당해 줌
할당된 공간 안에서 남는 메모리가 낭비되는 현상
### 1.1.2. 원인
운영체제가 메모리를 관리하기 쉽게 일정한 크기로 잘라서 할당하기 때문
## 1.2. 외부 단편화(External Fragmentation)
### 1.2.1. 상황
메모리 전체로 보면 빈 공간의 합계는 충분히 큼
하지만 공간들이 작게 흩어져 있어 큰 덩어리의 프로세스를 할당할 수 없음
### 1.2.2. 원인
서로 다른 크기의 프로세스들이 메모리에 들어왔다 나갔다를 반복
이러한 과정 중 중간에 작은 구멍들이 생기기 때문

---
# 2. 해결 방법
## 2.1. 페이징
메모리를 동일한 크기의 작은 조각으로 나누어 관리
외부 단편화는 해결되지만 내부 단편화 발생 가능
## 2.2. 세그멘테이션
프로세스를 논리적인 의미 단위(함수, 배열)로 나누어 할당
내부 단편화는 줄어들지만 외부 단편화가 심해질 수 있음
## 2.3. 메모리 풀
필요한 크기의 메모리 덩어리를 미리 만들어 놓고 빌려주는 방식
## 2.4. 압축
흩어진 빈 공간들을 한 쪽으로 몰아 큰 빈 공간을 만드는 작업
메모리에서는 오버헤드가 커서 잘 쓰지 않음
